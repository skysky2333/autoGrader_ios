Developer Guide

CanvasConnect Architecture

Module structure, pipeline design, Canvas API wrapper, matching algorithm, and extension points.

Overview

This guide describes the CanvasConnect codebase: a small Python package that turns HGrader exports and Canvas rosters into locked student matches, grade CSVs or API posts, and optional PDF uploads. Everything below maps to the implementation under CanvasConnect/canvas_connect/.

Architecture

CanvasConnect is fully open source and ships as a zero-dependency Python 3.11+ CLI built on the standard library only. Execution follows a pipeline: each step reads and writes artifacts under a shared, timestamped run directory.

models.py defines the core dataclasses used across parsing, matching, and API layers (LocalDataset, LocalSubmission, roster and match records, upload results). config.example.toml in the repo root documents every supported TOML key with inline comments; the loader in config.py mirrors those fields.

AspectDetail
LanguagePython 3.11+ (tomllib for config)
DependenciesNone — stdlib only (urllib, csv, json, dataclasses, difflib, unicodedata, …)
Entry pointcanvas-connect (shebang launcher) → cli.main
ConfigurationTOML via CanvasConnectConfig.load(); legacy single export_path is accepted if export_paths is omitted
HTTPurllib.request with Bearer auth and manual multipart uploads
Testingunittest under CanvasConnect/tests/

The CLI uses argparse with parser.add_subparsers(dest="command", required=True). A global --config path is passed to CanvasConnectConfig.load() (optional; missing file yields defaults).

_resolve_export_paths(config, cli_exports): if the CLI passes --export, those paths win; otherwise uses config.export_paths; if still empty, exits with SystemExit explaining that export_paths or --export is required.

models.py

LocalSubmission fields: submission_id, folder_name, student_name, total_score, max_score, teacher_reviewed, name_needs_review, created_at, overall_notes, scan_paths: list[str], pdf_path: str, grades: list[dict] (default empty).

LocalDataset fields: export_paths, title, created_at, question_count, total_points, submissions: list[LocalSubmission]. from_dict accepts legacy export_path for older JSON.

Also defined: CanvasStudent, MatchCandidate, MatchRecord, LockedMatchRecord, UploadResult, plus path_string() helper.

pipeline.py

Public step functions: make_run_dir, inspect_export_step, load_roster_step, match_step, review_matches_step, build_grade_csv_step, post_grades_step, upload_submissions_step, run_pipeline.

run_pipeline flow: make_run_dir → inspect_export_step → load_roster_step → match_step → optional confirm → review_matches_step → if gradebook_csv then build_grade_csv_step, else if grade_via_api then post_grades_step → unless skip_upload, upload_submissions_step → writes report.json with artifact paths.

Interactive review (review_matches_step / _prompt_for_record): for each MatchRecord, candidates are re-ranked with rank_candidates; the menu shows the top 5 by score. Choices: a digit to select (empty input defaults to 1), r rename local name and re-rank, s skip with optional note, u enter a Canvas user_id manually. Conflicts with another local submission already claiming the same user_id prompt for confirmation. Auto-accepted rows skip the menu unless --yes changes behavior for ambiguous rows.

merge_rosters: when API students exist, builds a user_id map from Canvas; for each CSV student with a matching user_id, fills empty section, sis_user_id, and login_id from the CSV. Returns CSV-only roster if the API list is empty; otherwise sorted merged API students.

validate_assignment_for_grading: requires the assignment lookup to return an id. validate_assignment_for_comment_workflow: delegates to validate_assignment_for_grading; used by the comment-attachment upload path. validate_assignment_for_upload: requires online_upload in assignment submission_types (retained for the legacy file-upload path).

build_execution_records: in test-student mode, replays one matched submission to the configured test_student_id by cloning a LockedMatchRecord with the test student's Canvas user ID and name. build_comment_text: assembles structured score breakdown text from a LocalSubmission and config toggles (total score, question scores, individual notes, overall notes).

export_parser.py

inspect_exports walks each export root: reads session-summary.json, then each submissions/*/summary.json with sibling scans/. Scan files are sorted with _page_sort_key using regex page-(\d+) in the filename; non-matching names sort after. build_submission_pdf from pdf.py writes per-student PDFs under inspect/pdfs/. ensure_unique_student_names enforces unique local names for the combined dataset. Outputs: inspect/local_dataset.json, inspect/local_submissions.csv, and inspect/session_csv_snapshots/ (via _write_session_csv_snapshots).

matching.py

score_name_match(left, right) -> (int 0–100, str reason): normalize_name applies NFKC, casefold, replaces non-alphanumeric runs with spaces; ascii_fold NFKD plus ASCII strip. Token-sort strings sort tokens alphabetically then join. Scoring: exact_normalized (100), exact_ascii (100), exact_token_set (98); else max of fuzzy_full, fuzzy_ascii, fuzzy_token_sort from difflib.SequenceMatcher ratios × 100 (rounded).

classify_match / _classify_match: let margin = top_score - (runner_up_score or 0). Status auto if (top_score > match_auto_accept_score and margin >= match_margin) or (top_score > match_review_floor and not name_needs_review and margin >= match_margin). Otherwise needs_review with reason name_needs_review if the export flagged the name, else the top candidate’s fuzzy reason. After building records, _flag_duplicate_auto_matches sets duplicate_candidate / duplicate_auto_match when multiple auto rows share the same matched_user_id.

api.py

CanvasAPI dataclass: base_url, token, timeout_seconds=60.

  • list_course_students — GET enrollments with pagination; follows Link header rel="next".
  • get_assignment / update_assignment — read/update assignment; form bodies are URL-encoded.
  • upload_submission_file — POST initiate to .../submissions/{user_id}/files with name/size/content_type; then _multipart_upload to Canvas storage with returned upload_params; JSON body must include file id.
  • upload_submission_comment_file — POST initiate to .../submissions/{user_id}/comments/files; same multipart flow as upload_submission_file but targets the comment attachment endpoint.
  • submit_file_on_behalf — POST .../submissions with online_upload, file_ids, user_id.
  • grade_or_comment_submission — PUT .../submissions/{user_id} with optional submission[posted_grade], comment[text_comment], and comment[file_ids][]; supports attaching comment files and updating grades in a single request.
  • grade_submission — PUT .../submissions/{user_id} with submission[posted_grade] and prefer_points_over_scheme=true.
  • update_assignment_grades — POST .../submissions/update_grades with grade_data[user_id][posted_grade] entries.
  • get_progress / wait_for_progress — poll /api/v1/progress/{id} until completed or failed, or timeout (default 300s poll window).

_request_json: Bearer Authorization, optional application/x-www-form-urlencoded body; on HTTPError, reads the body and raises CanvasAPIError with status and detail. _multipart_upload builds custom multipart/form-data with a UUID boundary; uses an opener with _NoRedirectHandler so redirects are not auto-followed—301/302/303 are handled by reading Location and issuing a follow-up GET. token_from_env(var) reads os.environ and raises CanvasAPIError if missing/blank.

CanvasAPIError subclasses RuntimeError.

csv_tools.py

load_gradebook_roster: opens CSV with UTF-8 BOM; DictReader rows skip blank names and rows whose name is Points Possible. Header aliases: student name (Student Name, Student), ID (Student ID, ID), SIS (SIS User ID), login (SIS Login ID, Login ID); Section column when present.

build_grade_import_csv: reads the template gradebook CSV, requires assignment_column in headers or raises ValueError listing available columns; matches locked rows by template Student ID / ID to final_user_id; writes scores into that column and preserves all other columns to output_path.

pdf.py

build_submission_pdf(scan_paths, output_path): if there is a single .pdf input, copies it with shutil.copyfile. For multiple images, only .jpg/.jpeg are supported. jpeg_info parses JPEG markers to read width, height, and component-derived colorspace (/DeviceGray, /DeviceRGB, /DeviceCMYK). _build_pdf_bytes emits minimal PDF 1.4 with one page per image: DCTDecode streams embed the raw JPEG bytes; content streams scale and paint each /ImN XObject; xref table and trailer complete the file.

Data Flow

  1. HGrader export — Folder tree with session-summary.json, submissions/<id>/summary.json, and scans/ images (and optional single PDF).
  2. Inspect — inspect_exports produces LocalDataset JSON, per-submission CSV snapshot, session CSV copies, and per-student PDFs under inspect/pdfs/.
  3. Canvas roster — API students (and optional gradebook CSV) merged to roster/roster.json (+ CSV mirror).
  4. Match — match_manifest.json / .csv with ranked candidates and statuses (auto, needs_review, unmatched, duplicate_candidate).
  5. Review — Interactive or --yes flow writes locked_manifest.json (and CSV): final user_id, names, SIS/login/section, skip/matched.
  6. Grades — Either gradebook/grade_import.csv from template + locked rows, or grades_api/grade_post_results.json from bulk (default) or per-user PUT in post_grades_step when bulk is disabled in code.
  7. Upload — uploads/upload_results.json (+ CSV) from comment-attachment API; assignment may be set to manual posting per config.
  8. Report — report.json at run root lists paths to the main artifacts produced in that run.

Matching Algorithm

  1. For each local submission, rank_candidates scores every roster name with score_name_match and sorts by descending score, then name, then user_id.
  2. The top few candidates are stored on each MatchRecord (three in the initial manifest; the interactive UI recomputes and can show five).
  3. Auto when the classifier accepts per match_auto_accept_score, match_review_floor, match_margin, and name_needs_review as implemented in _classify_match.
  4. Needs review when scores are ambiguous, margin is too small, or the export set name_needs_review.
  5. Duplicate detection — after auto classification, _flag_duplicate_auto_matches demotes rows that share the same auto user_id to duplicate_candidate.
  6. Manifests are written as JSON and CSV for audit and spreadsheet use.

Error Handling

MechanismWhen
CanvasAPIErrorHTTP failures from Canvas (including missing token via token_from_env), upload errors, multipart redirect issues, progress timeouts.
ValueErrorValidation: empty roster, missing Canvas fields for API steps, assignment not allowing uploads, published assignment when disallowed, missing assignment column in template CSV, empty export path list in parser, etc.
RuntimeErrorUser cancellation: run_pipeline confirm prompts, upload confirmation declined in upload_submissions_step.
FileNotFoundErrorMissing export directory, missing submissions folder, or missing per-export files expected during inspect.
SystemExitCLI resolution of export paths when neither --export nor config paths are set.
PDFError (pdf.py)Invalid or unsupported scan inputs when building submission PDFs.

Testing

cd CanvasConnect
python -m unittest discover -s tests

Tests live in CanvasConnect/tests/ and cover export parsing, matching, CSV helpers, pipeline behavior, and PDF assembly. Canvas HTTP is not required for the default unit suite.

Extension Points

  • Other LMS backends — mirror CanvasAPI and token handling for a different REST surface.
  • Matching — extend score_name_match or add pre-filters (e.g. email, SIS ID) before fuzzy name rank.
  • Exports — adjust export_parser.py if HGrader’s on-disk layout or summary.json fields change.
  • Automation — call run_pipeline or individual *_step functions from scripts; use --yes for non-interactive runs where appropriate.
