User Guide

CanvasConnect User Guide

Configure CanvasConnect, match students, post grades, and upload submissions to Canvas LMS.

Prerequisites

  • Python 3.11+ installed on your machine
  • Canvas API token — generate one from Canvas → Account → Settings → New Access Token
  • HGrader export folder(s) — extracted from the ZIP package produced by HGrader
  • Canvas assignment — an "On Paper" assignment with its posting policy set to manual (for comment-attachment uploads), or any type for grade-only posting

Configuration

CanvasConnect uses a TOML configuration file. Start by copying the example:

cd CanvasConnect
cp config.example.toml config.toml

Edit config.toml with your settings:

# Paths to extracted HGrader export folder(s)
export_paths = [
    "/path/to/exported/session-folder"
]

# Canvas LMS connection
canvas_base_url = "https://your-institution.instructure.com"
course_id = 12345
assignment_id = 67890

# Authentication — name of the env var holding your token
token_env_var = "CANVAS_API_TOKEN"

# Optional: column name in Canvas Gradebook CSV
# assignment_column = "Assignment 1 (12345)"

# Output directory for run artifacts
output_root = "output"

Set your Canvas API token as an environment variable:

export CANVAS_API_TOKEN="your-canvas-api-token"
Token security
Never commit your API token to version control. Use environment variables or a secrets manager.

Running the Full Pipeline

The simplest way to use CanvasConnect is the run command, which executes all steps in sequence:

./canvas-connect run

This will:

  1. Inspect and validate the export folder(s)
  2. Build submission PDFs from scan images
  3. Load the Canvas roster
  4. Match local students to Canvas users
  5. Present matches for interactive review
  6. Post grades to Canvas
  7. Upload PDF submissions (if configured)

All artifacts are saved to a timestamped run directory under output_root.

Step-by-Step Mode

For more control, run each command individually:

# 1. Inspect exports and build PDFs
./canvas-connect inspect-export

# 2. Load the Canvas roster
./canvas-connect load-roster

# 3. Match students
./canvas-connect match

# 4. Review matches interactively
./canvas-connect review-matches

# 5a. Build a Gradebook import CSV (CSV path)
./canvas-connect build-grade-csv

# 5b. OR post grades via API (API path)
./canvas-connect post-grades

# 6. Upload submission PDFs
./canvas-connect upload-submissions

Student Matching

CanvasConnect matches student names from HGrader exports to Canvas roster entries using fuzzy string matching:

How It Works

  1. Each local student name is compared against all Canvas roster names using sequence-based similarity scoring.
  2. Matches above the auto-accept threshold are accepted automatically.
  3. Matches above the review floor but below auto-accept are flagged for manual review.
  4. Matches below the review floor are marked as unmatched.

Interactive Review

The review-matches command presents each flagged match and lets you:

  • Accept the suggested match
  • Reject and leave unmatched
  • Rename the local student name and re-run matching
  • Manually select a different Canvas user

Tuning Thresholds

Matching thresholds are configurable in config.toml:

[matching]
auto_accept_score = 0.90    # Accept matches scoring above this
review_floor = 0.60         # Review matches between floor and auto_accept
margin = 0.10               # Minimum gap between best and second-best match

Grade Posting

CanvasConnect supports two methods for delivering grades to Canvas:

API Posting (Recommended)

The post-grades command posts grades directly via the Canvas REST API. It supports both individual submission updates and bulk grade posting with progress tracking.

Gradebook CSV

The build-grade-csv command generates a CSV file compatible with Canvas's Gradebook import. This is useful when you prefer to review and import grades manually, or when API access is restricted.

Gradebook CSV mode
To use the CSV path, you need to first download your course Gradebook from Canvas and set the assignment_column in your config to match the column header.

PDF Upload & Comment Posting

The upload-submissions command:

  1. Assembles a PDF for each student from their scanned images.
  2. Uploads the PDF as a submission comment attachment via the Canvas API.
  3. Posts a structured score breakdown as comment text (configurable).
  4. Optionally updates the grade in the same request.

The upload payload is configurable in config.toml:

  • upload_attach_pdf_as_comment — attach the scanned PDF
  • upload_post_grade — update the grade in the same request
  • upload_comment_enabled — post a text comment with score breakdown
  • upload_comment_include_total_score, upload_comment_include_question_scores, upload_comment_include_individual_notes, upload_comment_include_overall_notes — control comment content

Release Controls

By default, CanvasConnect enforces manual grade posting:

  • When enforce_manual_post_policy is true (default), the assignment is set to post grades manually so scores stay hidden until you explicitly post them in Canvas.
  • Use an On Paper assignment type for the comment-attachment workflow.

Test Mode

To verify the upload before running against the full class:

  • Set test_student_id in config.toml to a test student's Canvas user ID.
  • Run upload-submissions --test-student-only or post-grades --test-student-only.
  • Optionally set test_source_local_submission_id or pass --test-source-submission-id to replay a specific submission.

Troubleshooting

"Export path not found"

Ensure export_paths in config.toml points to the extracted folder, not the ZIP file. The folder must contain session-summary.json.

"Token not set"

Set the environment variable named in token_env_var (default: CANVAS_API_TOKEN). Verify with echo $CANVAS_API_TOKEN.

No matches found

Check that student names in HGrader match the names in Canvas. Use review-matches to manually correct mismatches. Lower the review_floor threshold if needed.

Upload fails with 401

Your API token may lack the required permissions. Ensure the token has course-level access and the assignment allows file uploads.

Bulk grade update times out

Large courses may take longer for Canvas to process bulk updates. CanvasConnect polls the progress endpoint automatically. If it times out, check Canvas's job queue.
